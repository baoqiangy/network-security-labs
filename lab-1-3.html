1. Lab Purpose

This lab teaches students how to inspect and document Linux network configuration. Students will use command-line tools to identify IP addresses, interfaces, MAC addresses, routes, DNS settings, listening services, and connectivity status in the isolated Kali–Ubuntu virtual lab environment.

Kali-Lab ───── Host-only / Private Network ───── Ubuntu-Target
Main goal: Students should be able to explain how a Linux system is connected to the network and prove that the virtual lab network is functioning correctly.

2. Learning Objectives

  1. Identify Linux network interfaces and IP addresses.
  2. Explain the difference between an IP address, subnet mask/prefix, MAC address, default route, and DNS server.
  3. Use ip addr, ip route, hostname, and uname to document system information.
  4. Use ping to test basic connectivity between Kali and Ubuntu.
  5. Use ss to view listening services and active connections.
  6. Use DNS tools and configuration files to inspect name resolution settings.
  7. Test service-level connectivity using SSH and curl.
  8. Save command output as evidence for a lab report.
  9. Explain OS-specific differences for Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

A network security analyst must confirm that two Linux systems are correctly connected before performing later security labs. The analyst needs to answer:

  1. What are the IP addresses of the two systems?
  2. Which network interfaces are active?
  3. Are both systems on the same isolated virtual network?
  4. Can Kali-Lab reach Ubuntu-Target?
  5. Which services are listening on Ubuntu?
  6. Can Kali connect to Ubuntu using SSH?
  7. How should the evidence be documented?

6. Safety Rules

  1. Use only the assigned Kali-Lab and Ubuntu-Target VMs.
  2. Keep both systems on the isolated host-only/private virtual network.
  3. Do not use bridged networking.
  4. Do not scan or test public systems, school networks, home routers, cloud systems, or classmates’ machines.
  5. Do not change the Windows or macOS host firewall.
  6. Do not submit screenshots containing passwords or sensitive personal information.
  7. Document only the traffic and configuration information generated inside this lab.

7. Key Concepts

IP Address

An IP address identifies a system on a network. In this lab, Kali and Ubuntu should each have an IP address on the same isolated virtual network.

Network Interface

A network interface is the virtual or physical adapter used to send and receive traffic. Linux interface names may include eth0, ens33, or enp0s3.

Prefix Length

The prefix length, such as /24, identifies the network portion of the IP address. For example, 192.168.56.102/24 usually means the system is on the 192.168.56.0 network.

Default Route

The default route tells a system where to send traffic when the destination is not on the local network. In an isolated host-only lab, a default route may be absent or may point to the virtual network gateway.

DNS

DNS translates names into IP addresses. In an isolated lab, DNS may be limited unless the VM is temporarily placed on NAT or Share with my Mac.

No IP address appearsNetwork adapter disabled or DHCP issueCheck VM network settings and restart networking or reboot the VM.
SSH connection refusedSSH server is not installed or running on UbuntuInstall/start openssh-server.
SSH times outWrong IP address, wrong network, or firewall issueVerify IP address and ping connectivity first.
