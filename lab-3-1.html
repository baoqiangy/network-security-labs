1. Lab Purpose

This lab introduces safe network discovery and host identification. Students will determine the local lab network range, identify live systems, compare manual discovery with Nmap discovery, examine neighbor/ARP information, and produce a small discovery report.

Kali-Lab ───── Host-only / Private Network ───── Ubuntu-Target
Main goal: Students should learn how to discover authorized lab hosts without scanning systems outside the approved virtual environment.

2. Learning Objectives

  1. Explain the purpose of host discovery in network security.
  2. Identify the local lab network range from IP address and prefix information.
  3. Use ping to verify reachability of a known host.
  4. Use ip neigh or arp to view local neighbor information.
  5. Use safe Nmap host-discovery options against the approved lab network only.
  6. Compare manual discovery, ARP/neighbor table evidence, and Nmap results.
  7. Identify IP addresses, MAC addresses, hostnames, and likely roles of discovered systems.
  8. Document scan scope, commands, findings, and ethical boundaries.
  9. Explain OS-specific differences across Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

A security analyst has been assigned to document a small internal lab network before later scanning and assessment activities. The analyst must determine which systems are alive, record their IP addresses, observe neighbor table entries, and document the authorized scope of discovery.

6. Safety Rules

  1. Perform discovery only inside the assigned Kali-Lab and Ubuntu-Target virtual network.
  2. Do not scan public IP addresses, school networks, home routers, cloud systems, or classmates’ computers.
  3. Do not use bridged networking for this lab.
  4. Keep both VMs on the isolated host-only/private network.
  5. Use only the discovery commands shown in this lab.
  6. Do not run vulnerability scans, exploit scripts, password attacks, or denial-of-service tools.
  7. Document scan scope before running Nmap.
  8. Stop and ask the instructor if the network range looks unfamiliar or appears to include non-lab systems.
Important: Nmap is a legitimate administration and security tool, but it must be used only on systems and networks where you have permission.

7. Key Concepts

Network Discovery

Network discovery is the process of identifying live hosts and basic network information within an authorized scope. It is commonly used before troubleshooting, inventory, monitoring, and security assessment.

Host Discovery vs. Port Scanning

ActivityPurposeExample
Host discoveryFind live systemsnmap -sn 192.168.56.0/24
Port scanningFind open services on a hostnmap -p 22 192.168.56.102

This lab focuses on host discovery and basic host identification. More detailed port scanning is handled in later labs.

ARP and Neighbor Tables

On a local network, systems use ARP to map IP addresses to MAC addresses. Linux displays these mappings with ip neigh.

Scan Scope

Scan scope defines exactly which systems or networks are approved for testing. A professional report should always state the approved scope before presenting discovery results.

Part 4 — Determine the Authorized Lab Network Range

On Kali-Lab, review the IP address and prefix length:

ip -brief addr
ip route

Look for an address such as:

192.168.56.101/24

In that example, the likely lab subnet is:

192.168.56.0/24

Create a scan-scope file on Kali. Replace the example subnet with your actual lab subnet:

cat > ~/Lab3-1-Discovery/scope/authorized-scope.txt <<'EOF'
Lab 3.1 Authorized Discovery Scope

Approved systems:
- Kali-Lab
- Ubuntu-Target

Approved network:
- REPLACE-WITH-YOUR-LAB-SUBNET, for example 192.168.56.0/24

Not approved:
- Public IP addresses
- School networks
- Home routers
- Cloud systems
- Classmates' machines

Discovery purpose:
- Identify live hosts inside the isolated lab network only.
EOF
nano ~/Lab3-1-Discovery/scope/authorized-scope.txt

Required Screenshot 4: Authorized scope file completed with the correct lab subnet.

nmap -sn 192.168.56.0/24

15. Troubleshooting Guide

ProblemLikely CauseFix

16. Instructor Notes

Core lesson: Network discovery should be accurate, limited, and authorized. Students should learn to define scope before running discovery commands.

Recommended Teaching Sequence

  1. Review what host discovery means and how it differs from port scanning.
  2. Have students identify their own lab subnet from ip addr and ip route.
  3. Require students to write the authorized scope before running Nmap.
  4. Use ping and neighbor-table evidence first.
  5. Run Nmap host discovery only inside the approved lab subnet.
  6. Compare discovery methods and discuss limitations.
  7. End with host inventory and ethical scope statement.

Recommended Short Version

  1. Record Kali and Ubuntu IP addresses.
  2. Determine the authorized lab subnet.
  3. Ping Ubuntu and review ip neigh.
  4. Run nmap -sn against the approved lab subnet.
  5. Run targeted discovery against Ubuntu only.
  6. Complete host inventory and final report.
