1. Lab Purpose

This lab introduces port scanning and service discovery with Nmap in a controlled and authorized virtual network. Students will scan only Ubuntu-Target, compare Nmap results with local service evidence, and explain how port scanning supports network security assessment.

Kali-Lab ───── Host-only / Private Network ───── Ubuntu-Target
Main goal: Students should learn how to identify exposed services responsibly and document what is visible from another host.

2. Learning Objectives

  1. Explain the purpose of port scanning and service discovery.
  2. Distinguish host discovery from port scanning.
  3. Run safe Nmap scans against a single approved target.
  4. Identify open, closed, and filtered port states.
  5. Use selected-port scanning to limit scope and reduce unnecessary traffic.
  6. Use service/version detection responsibly against the approved Ubuntu-Target VM.
  7. Compare Nmap results with local ss and systemctl evidence.
  8. Start and stop a temporary HTTP service to observe before-and-after scan results.
  9. Save scan output in normal, grepable, and XML-style formats when appropriate.
  10. Prepare a concise port scan report with scope, evidence, findings, and recommendations.

6. Safety Rules

  1. Scan only Ubuntu-Target unless the instructor explicitly authorizes another lab host.
  2. Do not scan public IP addresses, school networks, home routers, cloud systems, or classmates’ machines.
  3. Do not use bridged networking.
  4. Keep both VMs on the isolated host-only/private network.
  5. Do not run vulnerability scripts, exploit scripts, password attacks, or denial-of-service tools.
  6. Use selected-port scans before broader scans.
  7. State the approved scope in your report before presenting results.
  8. Stop the temporary HTTP service at the end of the lab.
Important: Port scanning must be authorized. This lab is limited to your own isolated virtual environment.

7. Key Concepts

Port Scanning

Port scanning identifies which network ports on a target respond to connection attempts or probes. It helps analysts understand what services may be exposed.

Service Discovery

Service discovery attempts to identify the application or protocol running on an open port. In Nmap, this is commonly done with -sV.

Common Nmap Port States

StateMeaningSecurity Interpretation
openA service appears to be listeningPotential service exposure
closedThe host responded, but no service is listening on that portHost reachable, service unavailable
filteredNmap cannot determine whether the port is open because packets may be blockedPossible firewall or filtering behavior
open|filteredNmap cannot confidently distinguish open from filteredOften seen with some UDP scans or filtered responses

Local View vs. Remote View

ViewToolWhat It Shows
Local service viewsudo ss -tulpenWhat Ubuntu is listening on locally
Service manager viewsystemctl status sshWhether a system service is running
Remote scan viewnmap from KaliWhat another host can see over the network
Application viewcurl or sshWhether the service works at the application level

15. Troubleshooting Guide

ProblemLikely CauseFix
Kali cannot ping UbuntuWrong IP address or VMs are on different virtual networksVerify ip -brief addr and correct VMware network mode.
Nmap is not installedPackage missing on KaliUse temporary NAT/Share with my Mac if allowed, then run sudo apt install -y nmap.
Port 22 is not openSSH is not installed, stopped, or filteredCheck systemctl status ssh and ss on Ubuntu.

Nmap finds unexpected hosts or portsWrong target or network modeStop scanning and verify you are using the Ubuntu IP only.

Apple Silicon package issueWrong architecture image or packageUse ARM64 Kali/Ubuntu images and repositories.

16. Instructor Notes

Core lesson: Port scanning should be scoped, purposeful, and evidence-based. Students should learn to connect Nmap’s remote view to local service evidence on the target.

Recommended Teaching Sequence

  1. Review authorized scope and ethical scanning rules.
  2. Verify the target by IP address and host discovery.
  3. Show local services on Ubuntu using ss.
  4. Run selected-port Nmap scans before broader scans.
  5. Run service/version detection against selected ports only.
  6. Start a temporary HTTP service and show before-and-after results.
  7. Save Nmap outputs in multiple formats.
  8. Have students compare local and remote evidence.
  9. Stop temporary services and complete the report.

Recommended Short Version

  1. Document scan scope.
  2. Verify Ubuntu is reachable.
  3. Run nmap -p 22,80,443,8080.
  4. Run nmap -sV on selected ports.
  5. Start HTTP on port 8080.
  6. Re-scan and compare results.
  7. Stop HTTP and verify cleanup.
  8. Submit tables and report.
