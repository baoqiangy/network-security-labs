1. Lab Purpose

The purpose of this lab is to help students understand the practical differences between symmetric and asymmetric encryption. Students will encrypt files with a shared secret, generate an asymmetric key pair, use a public key to encrypt a small message, use a private key to decrypt it, and create digital signatures to support authenticity and integrity.

Main goal: Students should learn that symmetric encryption is efficient for data, asymmetric encryption is useful for key exchange and identity-related operations, and digital signatures support integrity and authenticity.

2. Learning Objectives

  1. Explain the difference between symmetric and asymmetric encryption.
  2. Use OpenSSL to encrypt and decrypt a file with AES-256-CBC and PBKDF2.
  3. Verify that encrypted data is unreadable without the correct passphrase.
  4. Generate an RSA private/public key pair with OpenSSL.
  5. Protect private key files with restrictive file permissions.
  6. Use a public key to encrypt a small message and a private key to decrypt it.
  7. Create and verify a digital signature using SHA-256 and an RSA key pair.
  8. Demonstrate a simplified hybrid-encryption workflow.
  9. Use hashes to verify encrypted and decrypted file integrity.
  10. Explain OS-specific OpenSSL differences across Windows, macOS, Kali, and Ubuntu.

3. Lab Scenario

You are a security analyst preparing a secure file exchange procedure for a small organization. The organization needs to protect confidential files, verify file integrity, and confirm who signed an important message. Your task is to test symmetric encryption, asymmetric encryption, and digital signatures using OpenSSL in a controlled lab environment.

7. Key Concepts

Symmetric Encryption

Symmetric encryption uses the same secret key or passphrase to encrypt and decrypt data. It is efficient and commonly used for encrypting files, disks, and bulk data.

Asymmetric Encryption

Asymmetric encryption uses a key pair: a public key and a private key. The public key may be shared. The private key must be protected. Data encrypted with the public key can be decrypted with the matching private key.

Digital Signature

A digital signature is created with a private key and verified with the corresponding public key. It supports integrity, authenticity, and non-repudiation concepts.

Hybrid Encryption

Hybrid encryption combines both approaches: a symmetric key encrypts the data, and asymmetric encryption protects the symmetric key or secret. This is common because asymmetric encryption is not designed for large files.
