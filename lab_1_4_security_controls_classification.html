1. Lab Purpose

The purpose of this lab is to help students move beyond simply naming security controls. Students will observe real and simulated controls, classify each control by category and function, and explain how the control supports confidentiality, integrity, availability, or risk reduction.

Main goal: Students should learn that a single control can belong to more than one classification. For example, a firewall is technical and preventive, while logs are technical and detective.

2. Learning Objectives

  1. Define technical, administrative, and physical security controls.
  2. Differentiate preventive, detective, corrective, deterrent, compensating, and directive controls.
  3. Identify security controls already present in the Kali-Lab and Ubuntu-Target environment.
  4. Classify Linux file permissions, user accounts, firewall state, SSH settings, logs, backups, and snapshots.
  5. Map security controls to confidentiality, integrity, and availability.
  6. Explain why layered security requires multiple control types.
  7. Identify missing or weak controls in a small lab environment.
  8. Recommend control improvements using risk-based reasoning.
  9. Prepare a professional security control classification matrix.
  10. Explain differences across Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

You are helping a small academic department review the security posture of a new virtual cybersecurity lab. The department already has some controls, such as user accounts, file permissions, logs, virtual machine snapshots, SSH access, and firewall tools. Your task is to classify these controls and recommend improvements so the environment is better protected and easier to manage.

6. Safety Rules

  1. Use only your assigned Kali-Lab and Ubuntu-Target VMs.
  2. Do not modify production systems, school systems, home routers, or classmates’ devices.
  3. Do not collect or submit passwords, private keys, or password hashes.
  4. Do not disable security controls on real systems.
  5. When observing controls, document evidence but avoid unnecessary configuration changes.
  6. Only create temporary test files and folders inside your lab directory.
  7. Keep the lab network host-only/private unless the instructor authorizes temporary Internet access.
Important: This lab is about classification and documentation. It is not an exploitation lab.

7. Key Concepts

Control Categories

CategoryMeaningExamples
Technical controlImplemented through hardware, software, or configurationFirewall, SSH keys, file permissions, encryption, logs
Administrative controlImplemented through policies, procedures, training, or governancePassword policy, acceptable use policy, incident response plan
Physical controlProtects physical access to systems, facilities, or equipmentLocks, cameras, badges, cable locks, server room access control

Control Functions

FunctionMeaningExamples
PreventiveStops or reduces the chance of an unwanted eventFirewall rule, file permission, MFA, locked door
DetectiveIdentifies or alerts about an eventLogs, IDS alert, auditd rule, security camera recording
CorrectiveRestores or fixes after an eventBackup restore, patch, incident response action
DeterrentDiscourages unwanted behaviorWarning banner, visible camera, policy notice
CompensatingAlternative control used when the preferred control is not feasibleManual review when automated monitoring is unavailable
DirectiveGuides behavior through rules or instructionsPolicy, procedure, configuration standard

Controls and the CIA Triad

Controls may protect confidentiality, integrity, availability, or more than one CIA element. For example, backups primarily support availability but may also support integrity when they restore a known-good version of data.

Lab 1.4 Control Classification Reference

Control Categories:
- Technical: implemented using technology, software, hardware, or configuration
- Administrative: implemented using policy, procedure, training, governance, or documentation
- Physical: implemented using physical protection of systems, facilities, or people

Control Functions:
- Preventive: helps stop an event before it happens
- Detective: helps identify that an event happened
- Corrective: helps fix or restore after an event
- Deterrent: discourages unwanted behavior
- Compensating: alternative control used when the preferred control is not feasible
- Directive: tells people what to do or how to behave

CIA Mapping:
- Confidentiality: protects information from unauthorized access
- Integrity: protects accuracy and trustworthiness
- Availability: keeps systems and data accessible when needed

16. Instructor Notes

Core lesson: Control classification builds vocabulary for later topics. Students should see that real security programs require a mix of technical, administrative, and physical controls, and each control should be connected to risk and the CIA triad.

Recommended Teaching Sequence

  1. Review control categories and control functions.
  2. Ask students to classify familiar examples before touching the VMs.
  3. Have students collect technical control evidence from Ubuntu.
  4. Have students classify administrative and physical control scenarios.
  5. Discuss why controls overlap across categories and functions.
  6. Have students complete a 12-control matrix and layered defense summary.
