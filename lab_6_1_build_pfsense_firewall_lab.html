Lab 6.1 — Build a pfSense Firewall Lab

Network Security hands-on lab based on the Lab 1.1 virtual environment. Students add a pfSense firewall VM, connect it between a WAN/NAT network and an isolated LAN network, configure interfaces, enable LAN services, test client connectivity, review default firewall behavior, and document OS-specific virtualization differences.

1. Lab Purpose

The purpose of this lab is to build a small virtual firewall environment using pfSense. Students will install or prepare a pfSense VM, connect one interface to a WAN/NAT network and one interface to an isolated LAN network, place Kali-Lab and Ubuntu-Target behind the firewall, and verify that traffic flows through pfSense.

Main goal: Students should understand the basic role of a firewall/router: separate networks, route traffic, apply firewall rules, provide DHCP/DNS services when configured, and log traffic for review.

2. Learning Objectives

  1. Describe the purpose of a firewall/router in a small network.
  2. Create a safe virtual topology with separate WAN and LAN networks.
  3. Install or prepare a pfSense firewall VM in VMware.
  4. Assign pfSense WAN and LAN interfaces correctly.
  5. Configure pfSense LAN addressing and optional DHCP service.
  6. Connect Kali-Lab and Ubuntu-Target to the pfSense LAN.
  7. Access the pfSense WebGUI from the LAN side.
  8. Verify client IP addressing, default gateway, DNS, and internet reachability.
  9. Review default LAN-to-WAN firewall behavior.
  10. Document important differences between Windows, Intel Mac, Apple Silicon Mac, Kali, Ubuntu, and pfSense.

3. Lab Scenario

Your organization wants a small lab network where security analysts can test hosts behind a dedicated firewall. You are asked to build the first version of this lab using pfSense as the firewall/router. The firewall must sit between an external WAN/NAT network and an internal isolated LAN. Kali-Lab and Ubuntu-Target should be placed behind pfSense so later labs can test firewall rules, NAT, logging, and segmentation.

6. Safety Rules

  1. Use only isolated virtual networks created for the lab.
  2. Do not bridge the pfSense LAN to a real production, school, or home network.
  3. Do not let pfSense DHCP serve addresses to any real network.
  4. Do not scan or attack systems outside the virtual lab.
  5. Take snapshots before major firewall or network changes.
  6. Record interface assignments carefully before applying changes.
  7. Do not expose pfSense WebGUI to the WAN interface.
  8. Change default pfSense credentials during setup if prompted.
Important: A misconfigured virtual firewall can disrupt your lab network or expose services unexpectedly. Keep the LAN private and verify each interface before connecting clients.

7. Key Concepts

Firewall

A firewall controls traffic between networks based on rules. In this lab, pfSense separates the WAN/NAT side from the internal LAN side.

Router

A router forwards traffic between networks. pfSense will route traffic from Kali-Lab and Ubuntu-Target on the LAN to the WAN/NAT network.

NAT

Network Address Translation allows private LAN clients to share the firewall WAN address when reaching outside networks.

WAN and LAN

The WAN side faces the upstream network. The LAN side faces protected internal clients. In this lab, the WAN connects to VMware NAT and the LAN connects to an isolated internal network.

Firewall Rules

pfSense firewall rules are applied on interfaces. A common default behavior is to allow LAN clients out while blocking unsolicited inbound traffic from the WAN side.

Part 2 — Create or Identify the WAN and LAN Virtual Networks

Windows with VMware Workstation Pro

  1. Open Virtual Network Editor as administrator.
  2. Confirm VMnet8 is configured as NAT. This will be the pfSense WAN.
  3. Create or identify a separate host-only network such as VMnet2. This will be the pfSense LAN.
  4. For the pfSense LAN VMnet, avoid running two DHCP servers. Either disable VMware DHCP for VMnet2 or plan to disable pfSense DHCP. This lab assumes pfSense DHCP will be used, so VMware DHCP should be disabled on the LAN VMnet if possible.

Intel Mac with VMware Fusion Pro

  1. Use Share with my Mac as the WAN/NAT network.
  2. Create or select a separate Private to my Mac network for the pfSense LAN.
  3. Make sure Kali-Lab and Ubuntu-Target will be attached to the same pfSense LAN network.

Apple Silicon Mac with VMware Fusion Pro

  1. Confirm with your instructor whether a compatible pfSense option is available.
  2. If no compatible pfSense VM is available, complete the design, topology, and verification worksheet portions using an instructor-provided remote pfSense appliance or alternate firewall VM.
  3. Do not attempt to boot an x86-64 pfSense ISO in an ARM-only virtual machine.

Part 5 — Assign pfSense Interfaces

At the pfSense console, assign interfaces if prompted or if the assignments are incorrect.

  1. Choose the console option for interface assignment if needed.
  2. Assign the WAN interface to the NIC connected to the VMware NAT network.
  3. Assign the LAN interface to the NIC connected to the isolated pfSense LAN network.
  4. Do not configure VLANs for this introductory lab unless instructed.

Typical interface names may look like em0, em1, vtnet0, vtnet1, or similar depending on the VM hardware type.
