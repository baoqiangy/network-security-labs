1. Lab Purpose

This lab introduces asymmetric cryptography and digital signatures. Students use GPG to create key pairs, export and import public keys, encrypt a file for another user, decrypt a file with a private key, sign a message, and verify the signature using the sender's public key.

Kali-Lab ───── Host-only / Private Network ───── Ubuntu-Target
Main goal: Students should understand that public keys can be shared, private keys must be protected, encryption provides confidentiality, and digital signatures provide integrity and sender verification.

2. Learning Objectives

  1. Explain the difference between symmetric and asymmetric encryption.
  2. Describe the roles of public keys and private keys.
  3. Generate a GPG key pair on Kali-Lab and Ubuntu-Target.
  4. Export and import public keys between systems.
  5. Encrypt a file for a recipient using the recipient's public key.
  6. Decrypt a file using the recipient's private key.
  7. Create a digital signature for a file.
  8. Verify a digital signature using the signer's public key.
  9. Explain how encryption, hashing, and digital signatures work together.
  10. Document key fingerprints, encryption evidence, decryption evidence, and signature verification results.
  11. Explain OS-specific differences across Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

A security analyst on Kali-Lab needs to send a confidential message to an administrator on Ubuntu-Target. The analyst encrypts the file using the administrator's public key. The administrator decrypts it using the private key. The analyst then signs a separate message, and the administrator verifies the signature to confirm that the file came from the expected sender and was not changed.

7. Key Concepts

Asymmetric Encryption

Asymmetric encryption uses a key pair. A public key can be shared with others, while the private key must be protected. A file encrypted with a recipient's public key can be decrypted only with the matching private key.

Digital Signatures

A digital signature is created with the sender's private key and verified with the sender's public key. It helps prove that a message came from the expected signer and has not been modified after signing.

Encryption vs. Signing

FunctionKey UsedPrimary Security Goal
Encrypt for a recipientRecipient's public keyConfidentiality
Decrypt received dataRecipient's private keyRecover plaintext
Sign a fileSender's private keyIntegrity, authentication, nonrepudiation support
Verify a signatureSender's public keyConfirm signer and file integrity

15. Troubleshooting Guide

16. Instructor Notes

Core lesson: Asymmetric cryptography solves key-sharing problems differently from symmetric encryption. Public keys are shared; private keys are protected. Encryption provides confidentiality, while signatures provide integrity and sender verification.

Recommended Teaching Sequence

  1. Review symmetric vs. asymmetric encryption.
  2. Generate Ubuntu's recipient key pair.
  3. Export Ubuntu's public key and import it on Kali.
  4. Encrypt a message on Kali for Ubuntu.
  5. Decrypt the message on Ubuntu.
  6. Generate Kali's signing key pair.
  7. Export Kali's public key and import it on Ubuntu.
  8. Sign a file on Kali and verify it on Ubuntu.
  9. Modify the signed file and show signature verification failure.
  10. Complete the final report.
