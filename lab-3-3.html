1. Lab Purpose

This lab introduces service enumeration and banner grabbing in a controlled environment. Students will use Nmap, curl, Netcat, and local Linux tools to identify what services are running, what version or banner information is visible, and why exposed service information matters to defenders.

Kali-Lab ───── Host-only / Private Network ───── Ubuntu-Target
Main goal: Students should learn how to collect basic service evidence responsibly and explain what information is exposed by common services.

2. Learning Objectives

  1. Explain the difference between port scanning and service enumeration.
  2. Use Nmap service/version detection against an approved target.
  3. Use curl to inspect HTTP headers and response behavior.
  4. Use nc to manually collect simple service banners from approved lab services.
  5. Compare automated enumeration results with manual banner evidence.
  6. Use ss and systemctl on Ubuntu to validate local service state.
  7. Start and stop a temporary HTTP service for controlled enumeration.
  8. Explain why service banners and headers can create information exposure.
  9. Document findings, scope, evidence, and recommendations in a professional report.
  10. Explain differences across Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

A security analyst has identified that an internal Ubuntu system is reachable. The next step is to determine what service information is visible from the network. The analyst must enumerate only the approved Ubuntu-Target VM, collect service banners or headers, validate findings locally, and recommend ways to reduce unnecessary information exposure.

6. Safety Rules

  1. Enumerate only Ubuntu-Target in your isolated lab environment.
  2. Do not enumerate public systems, school networks, home routers, cloud systems, or classmates’ machines.
  3. Do not use bridged networking.
  4. Do not use vulnerability scripts, exploit scripts, brute-force tools, or denial-of-service tools.
  5. Use only selected ports assigned in this lab: TCP/22 and TCP/8080.
  6. Do not submit passwords or sensitive information in screenshots or reports.
  7. Stop temporary services at the end of the lab.
  8. Document the approved scope before collecting service information.
Important: Banner grabbing and service enumeration are allowed only in this authorized lab environment.

7. Key Concepts

Service Enumeration

Service enumeration is the process of identifying application-level details about services running on open ports. It goes beyond simply knowing that a port is open.

Banner Grabbing

Banner grabbing collects information that a service reveals during a connection. A banner may include protocol information, server software, version hints, or response headers.

Automated vs. Manual Enumeration

ApproachToolWhat It Shows
Automatednmap -sVDetected service name and possible version
HTTP-specificcurl -I, curl -vHTTP status line, headers, and response behavior
ManualncRaw banner or protocol response
Local validationss, systemctlWhat is actually running on the target

16. Instructor Notes

Core lesson: Enumeration is about interpreting service information, not merely collecting tool output. Students should connect each finding to local validation and security meaning.

Recommended Teaching Sequence

  1. Review scope and authorized target.
  2. Verify SSH and start temporary HTTP on Ubuntu.
  3. Run Nmap service detection on TCP/22 and TCP/8080.
  4. Collect HTTP headers with curl -I.
  5. Collect manual HTTP and SSH banners with Netcat.
  6. Compare automated and manual findings.
  7. Discuss information exposure and defensive recommendations.
  8. Stop the temporary HTTP service and complete the report.
