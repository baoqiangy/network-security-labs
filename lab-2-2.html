1. Lab Purpose

This lab introduces DNS and name resolution from a practical network security perspective. Students will inspect how Linux systems resolve names, compare local host-file resolution with DNS-based lookup, test name-based connectivity, and document why name resolution is important for troubleshooting and security monitoring.

Kali-Lab ───── Host-only / Private Network ───── Ubuntu-Target
Main goal: Students should understand that computers communicate using IP addresses, but users and applications often rely on names. Name resolution connects the two.

2. Learning Objectives

  1. Explain the role of DNS in network communication.
  2. Describe the difference between local host-file resolution and DNS server resolution.
  3. Inspect Linux name-resolution settings using /etc/hosts, /etc/resolv.conf, and resolvectl.
  4. Use dig and nslookup to query DNS records when Internet or DNS access is available.
  5. Create a local hostname for Ubuntu-Target on Kali-Lab.
  6. Test name-based connectivity using ping, ssh, and curl.
  7. Compare successful and failed name-resolution behavior.
  8. Document DNS troubleshooting evidence using screenshots and saved command output.
  9. Explain OS-specific differences across Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

A network security analyst is troubleshooting a small lab network. The analyst can connect to Ubuntu-Target by IP address but wants to connect by hostname. The analyst must determine how names are resolved, create a local name record, verify that the name works, and explain what happens when DNS or local host records are incorrect.

7. Key Concepts

DNS

Domain Name System, or DNS, translates human-readable names into IP addresses. For example, a name such as example.com can resolve to an IP address used by computers.

Local Host File

The /etc/hosts file can define local hostname-to-IP mappings. These local mappings can work even when Internet DNS is unavailable.

Resolver Configuration

The resolver tells the system where to send DNS queries. On Linux, useful places to inspect include /etc/resolv.conf and resolvectl status.

Forward and Reverse Lookup

Lookup TypeMeaningExample
Forward lookupName to IP addressubuntu-target.local → 192.168.56.102
Reverse lookupIP address to name192.168.56.102 → ubuntu-target.local
