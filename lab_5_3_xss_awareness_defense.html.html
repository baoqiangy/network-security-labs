1. Lab Purpose

The purpose of this lab is to help students understand Cross-Site Scripting, or XSS, as a web application output-handling weakness. Students will use a local-only training application to observe how untrusted input can be rendered unsafely and how output encoding, server-side validation, Content Security Policy, and secure cookie settings reduce risk.

Main goal: Students should learn that XSS defense is primarily about treating user-controlled data as untrusted when it is displayed, not just about blocking certain words or characters.

2. Learning Objectives

  1. Define reflected XSS and stored XSS at a conceptual level.
  2. Explain why untrusted input must be encoded before display.
  3. Build and run a local Flask XSS awareness application on Ubuntu-Target.
  4. Use safe local proof-of-concept input to observe weak rendering behavior.
  5. Compare weak rendering with output-encoded rendering.
  6. Demonstrate how Content Security Policy can reduce script execution risk.
  7. Review secure cookie attributes such as HttpOnly, SameSite, and Secure.
  8. Document XSS risks, limitations, and remediation strategies.
  9. Collect evidence using curl, browser screenshots, and application logs.
  10. Explain OS-specific differences across Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu environments.

3. Lab Scenario

A development team created a feedback feature for an internal web application. Early testing shows that user comments may be displayed without proper output encoding. Your task is to test the issue safely in a local lab, compare vulnerable and defended behavior, and recommend practical controls for preventing XSS in production applications.

6. Safety Rules

  1. Use only the local training app created in this lab.
  2. Do not test XSS on public websites, school systems, classmates’ systems, or production systems.
  3. Do not steal cookies, tokens, sessions, personal data, or credentials.
  4. Do not send data to external servers or use payloads that exfiltrate information.
  5. Use only benign proof-of-concept strings, such as a local alert or text-rendering demonstration.
  6. Keep the virtual network host-only/private.
  7. Do not store real sensitive information in the training app.
  8. Stop the Flask app after the lab unless the instructor says to keep it running.
Important: This is a defensive awareness and mitigation lab. The purpose is to understand and prevent XSS, not to attack real users or real applications.

7. Key Concepts

Cross-Site Scripting

Cross-Site Scripting occurs when a web application displays untrusted input in a way that allows the browser to interpret it as active content. This can affect confidentiality, integrity, session safety, and user trust.

Reflected XSS

Reflected XSS occurs when input from a request is immediately included in the response without safe handling.

Stored XSS

Stored XSS occurs when input is saved by the application and later displayed to users without safe handling.

Output Encoding

Output encoding converts special characters into safe text before displaying them. For HTML contexts, characters such as < and > should be encoded so the browser treats them as text.

Content Security Policy

Content Security Policy, or CSP, is an HTTP response header that can restrict what scripts, styles, images, and other resources a browser may load or execute. CSP is a defense-in-depth control and should not replace output encoding.

from flask import Flask, request, make_response, redirect
import html
import logging

app = Flask(__name__)
comments = []

logging.basicConfig(
    filename="xss_awareness_lab.log",
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)

def page(title, body, extra_headers=None):
    response = make_response(
        "<!DOCTYPE html><html><head><title>" + html.escape(title) +
        "</title></head><body><h1>" + html.escape(title) + "</h1>" +
        body + "<hr><p><a href='/'>Home</a></p></body></html>"
    )
    if extra_headers:
        for key, value in extra_headers.items():
            response.headers[key] = value
    return response

@app.route("/")
def home():
    body = """
    <ul>
      <li><a href="/weak-reflect?q=hello">Weak reflected rendering</a></li>
      <li><a href="/safe-reflect?q=hello">Safe reflected rendering</a></li>
      <li><a href="/weak-comments">Weak stored comment demo</a></li>
      <li><a href="/safe-comments">Safe stored comment demo</a></li>
      <li><a href="/csp-demo?q=hello">CSP demo</a></li>
      <li><a href="/cookie-demo">Cookie demo</a></li>
    </ul>
    <p>This app is for local defensive training only.</p>
    """
    return page("Lab 5.3 XSS Awareness and Defense", body)

@app.route("/weak-reflect")
def weak_reflect():
    q = request.args.get("q", "")
    logging.info("weak-reflect q=%r", q)
    return page("Weak Reflected Rendering", "<p>Search term: " + q + "</p>")

@app.route("/safe-reflect")
def safe_reflect():
    q = request.args.get("q", "")
    logging.info("safe-reflect q=%r", q)
    return page("Safe Reflected Rendering", "<p>Search term: " + html.escape(q) + "</p>")

@app.route("/weak-comments", methods=["GET", "POST"])
def weak_comments():
    if request.method == "POST":
        comment = request.form.get("comment", "")
        comments.append(comment)
        logging.info("weak-comments stored comment=%r", comment)
        return redirect("/weak-comments")
    comment_html = "".join(["<li>" + c + "</li>" for c in comments])
    body = "<form method='post'><input name='comment' size='60'><button>Submit</button></form><ul>" + comment_html + "</ul>"
    return page("Weak Stored Comment Demo", body)

@app.route("/safe-comments", methods=["GET", "POST"])
def safe_comments():
    if request.method == "POST":
        comment = request.form.get("comment", "")
        comments.append(comment)
        logging.info("safe-comments stored comment=%r", comment)
        return redirect("/safe-comments")
    comment_html = "".join(["<li>" + html.escape(c) + "</li>" for c in comments])
    body = "<form method='post'><input name='comment' size='60'><button>Submit</button></form><ul>" + comment_html + "</ul>"
    return page("Safe Stored Comment Demo", body)

@app.route("/csp-demo")
def csp_demo():
    q = request.args.get("q", "")
    headers = {
        "Content-Security-Policy": "default-src 'self'; script-src 'self'; object-src 'none'; base-uri 'self'; frame-ancestors 'none'",
        "X-Content-Type-Options": "nosniff",
        "X-Frame-Options": "DENY"
    }
    return page("CSP Demo", "<p>Input displayed with encoding: " + html.escape(q) + "</p>", headers)

@app.route("/cookie-demo")
def cookie_demo():
    response = page("Cookie Demo", "<p>Review Set-Cookie headers with curl -I.</p>")
    response.set_cookie("lab_session", "local-demo-value", httponly=True, samesite="Strict")
    response.set_cookie("display_pref", "lab-light-mode", httponly=False, samesite="Lax")
    return response

@app.route("/clear")
def clear():
    comments.clear()
    logging.info("comments cleared")
    return redirect("/")

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5001)
EOF
