Week 1

Security Foundations, Lab Safety, CIA Triad, and Controls

This week establishes safe lab practices and gives students a foundation before scanning, exploitation, logging, or incident response activities.
1.1
Build an Isolated Cybersecurity Lab
Skills: Create host-only/internal networks, install Ubuntu/Kali, verify no Internet exposure for attack labs
Deliverable: Network diagram and VM connectivity screenshots
1.2
Asset Inventory and Security Baseline
Skills: Identify OS version, users, open ports, installed services, patch status
Deliverable: Baseline inventory worksheet
1.3
CIA Triad Walkthrough
Skills: Demonstrate confidentiality, integrity, and availability using simple file permissions, hashing, and service availability
Deliverable: Screenshots showing permission control, hash verification, and service status
1.4
Security Controls Classification
Skills: Match technical, administrative, physical, preventive, detective, and corrective controls to realistic scenarios
Deliverable: Control classification table
Week 2

Networking Fundamentals, Protocol Analysis, and Attack Surfaces

Suggested tools: Wireshark, Nmap, Ubuntu, Windows VM, UFW.
2.1
Packet Capture with Wireshark
Skills: Capture ARP, ICMP, DNS, TCP handshake, HTTP traffic
Deliverable: Annotated packet screenshots
2.2
Network Mapping with Nmap
Skills: Scan only student-owned lab VMs; identify hosts, ports, services, and OS guesses
Deliverable: Nmap results and explanation
2.3
Service Exposure and Attack Surface Review
Skills: Compare open services before and after disabling unnecessary services
Deliverable: Before/after exposure report
2.4
Host Firewall Configuration
Skills: Configure UFW or Windows Firewall to allow/deny specific services
Deliverable: Firewall rule table and test results
Week 3

Authentication, Authorization, Passwords, and Access Control

Safety note: Do not have students attack real accounts or real services. Password testing should use instructor-created sample hashes or toy examples only.
3.1
Linux User and Group Access Control
Skills: Create users, groups, file permissions, and sudo restrictions
Deliverable: Permission matrix and test screenshots
3.2
SSH Hardening
Skills: Disable root login, change SSH settings, use SSH keys, restrict password login
Deliverable: Hardened SSH configuration summary
3.3
Password Policy and Offline Hash Awareness
Skills: Create weak and strong passwords, examine hash storage concepts, test password strength using safe offline examples
Deliverable: Password policy recommendation
3.4
Role-Based Access Control Scenario
Skills: Design RBAC for a small organization: admin, help desk, student, guest, auditor
Deliverable: RBAC table and justification
Week 4

Cryptography, Hashing, PKI, Certificates, and Secure Communications

Suggested tools: OpenSSL, Apache/Nginx, Ubuntu Server.
4.1
Hashing and File Integrity
Skills: Generate SHA-256 hashes; detect file tampering
Deliverable: Hash comparison report
4.2
Symmetric and Asymmetric Encryption with OpenSSL
Skills: Encrypt/decrypt files; compare shared-key and public/private-key methods
Deliverable: Command log and explanation
4.3
Create a Private Certificate Authority
Skills: Generate CA key, server certificate, and signed certificate
Deliverable: Certificate chain screenshots
4.4
Enable HTTPS on a Local Web Server
Skills: Configure Apache or Nginx with TLS using a lab certificate
Deliverable: Browser/certificate screenshots and configuration summary
Week 5

Threats, Vulnerabilities, Web Attacks, and Mitigation

Safety note: Keep all web attack labs inside DVWA, Juice Shop, or another intentionally vulnerable local application. Do not scan or test public websites.
5.1
Vulnerability Scanning in a Local Lab
Skills: Scan DVWA/Juice Shop or a purposely vulnerable VM using safe local tools
Deliverable: Vulnerability scan report
5.2
Web Application Input Validation Lab
Skills: Demonstrate how poor input validation affects a test application, then apply a basic mitigation
Deliverable: Before/after screenshots
5.3
XSS Awareness and Defense
Skills: Use DVWA/Juice Shop to observe reflected/stored XSS behavior in a controlled setting; explain defenses
Deliverable: XSS mitigation worksheet
5.4
Patch and Hardening Verification
Skills: Apply updates, disable risky services, rescan, and compare results
Deliverable: Before/after risk reduction report
Week 6

Secure Network Architecture, Segmentation, Firewalls, DMZs, and VPNs

Suggested tools: pfSense CE, Ubuntu Server, Kali/Ubuntu client, WireGuard or OpenVPN.
6.1
Build a pfSense Firewall Lab
Skills: Configure WAN/LAN interfaces, routing, DHCP, and basic firewall rules
Deliverable: pfSense topology diagram
6.2
Design and Test a DMZ
Skills: Place a web server in a DMZ and restrict access from LAN and WAN
Deliverable: Firewall rule table and test results
6.3
Network Segmentation and Least Privilege
Skills: Create separate subnets for user, server, and admin zones; test allowed/blocked traffic
Deliverable: Segmentation policy matrix
6.4
Site-to-Site or Client VPN Lab
Skills: Configure WireGuard or OpenVPN in the lab environment
Deliverable: VPN connection proof and security explanation
Week 7

Security Monitoring, Logging, IDS, and SIEM

This week connects strongly to Security Operations and prepares students for incident response.
7.1
Linux Logging and Auditd
Skills: Enable auditd rules for file access, privilege use, and authentication events
Deliverable: Log samples and interpretation
7.2
Network IDS with Suricata
Skills: Install Suricata, generate safe test traffic, review IDS alerts
Deliverable: Alert screenshots and explanation
7.3
Wazuh Agent and SIEM Dashboard
Skills: Install Wazuh manager/agent, connect endpoint logs, view alerts
Deliverable: Wazuh dashboard screenshots
7.4
Alert Triage Exercise
Skills: Review a small set of alerts and classify them as true positive, false positive, low priority, or high priority
Deliverable: Alert triage worksheet
Week 8

Cloud, Virtualization, Endpoint Security, Data Protection, and Backup

Suggested tools: Docker, Ubuntu, Windows Security settings, local backup tools, cloud sandbox if available.
8.1
Cloud IAM Policy Review
Skills: In a cloud sandbox or simulated IAM worksheet, compare least privilege vs. overly permissive access
Deliverable: IAM policy analysis
8.2
Container Security Basics
Skills: Run a Docker container, inspect image metadata, identify risks of running as root
Deliverable: Container security checklist
8.3
Endpoint Hardening Checklist
Skills: Apply endpoint protections: updates, firewall, disk encryption discussion, antivirus/EDR settings where available
Deliverable: Endpoint hardening report
8.4
Backup, Restore, and Ransomware Resilience Simulation
Skills: Create backup, modify/delete files, restore from backup, document recovery steps
Deliverable: Backup and recovery evidence
Week 9

Incident Response, Risk Management, Governance, and Capstone

9.1
Incident Response Plan Development
Skills: Create a small-business IR plan with preparation, detection, containment, eradication, recovery, and lessons learned
Deliverable: IR plan
9.2
Digital Evidence Collection Basics
Skills: Collect logs, screenshots, hashes, timestamps, and write a basic chain-of-custody form
Deliverable: Evidence package
9.3
Risk Register Using NIST-Style Thinking
Skills: Identify assets, threats, vulnerabilities, likelihood, impact, risk level, and mitigation
Deliverable: Risk register
9.4
Final Network Defense Capstone
Skills: Students secure a small network, document controls, monitor logs, and present findings
Deliverable: Final report and short demo video
