1. Overview

This guide explains how to install VMware on Windows and macOS so students can complete the Network Security labs using Kali Linux and Ubuntu virtual machines.

Host Computer VMware Product Recommended Lab Network Typical VM Architecture
Windows laptop or desktop VMware Workstation Pro VMnet1 Host-only x86_64
Intel MacBook VMware Fusion Pro Private to my Mac x86_64
Apple Silicon MacBook VMware Fusion Pro Private to my Mac ARM64 / aarch64
Important: For cybersecurity labs, students should use host-only/private networking unless the instructor specifically authorizes Internet access or bridged networking.

2. Installing VMware Workstation Pro on Windows

Product to Install

Windows students should install:

VMware Workstation Pro

Part 1 — Check Windows System Requirements

Before installation, confirm the following:

  1. The computer is running Windows and the student has administrator access.
  2. Hardware virtualization is enabled in BIOS/UEFI.
  3. The computer has enough resources for two virtual machines.
Resource Recommended Minimum Workable
RAM 16 GB or more 8 GB
Free disk space 80–100 GB 50 GB
CPU Modern Intel or AMD CPU with virtualization support Virtualization-capable CPU
Permissions Administrator account Administrator password available

Part 2 — Download VMware Workstation Pro

  1. Open a web browser.
  2. Go to the Broadcom Support Portal.
  3. Sign in with a Broadcom account. Create a free account if needed.
  4. Search for:
VMware Workstation Pro
  1. Select the latest available version for Windows.
  2. Accept any required terms and conditions.
  3. Download the Windows installer.

The installer filename is usually similar to:

VMware-workstation-full-17.x.x-xxxxxxx.exe

Part 3 — Install VMware Workstation Pro

  1. Open the Downloads folder.
  2. Find the VMware Workstation installer.
  3. Right-click the installer.
  4. Select Run as administrator.
  5. If Windows asks for permission, click Yes.
  6. When the setup wizard opens, click Next.
  7. Accept the license agreement.
  8. Choose Typical installation unless the instructor tells you otherwise.
  9. Click Install.
  10. Wait for the installation to complete.
  11. Click Finish.
  12. Restart Windows if prompted.

Part 4 — Recommended Setup Choices

Installation Option Recommended Choice Reason
Enhanced Keyboard Driver Install Improves keyboard handling inside virtual machines
Check for Product Updates Yes Allows update notifications
Customer Experience Improvement Program Optional Not required for class labs
Desktop Shortcut Yes Makes VMware easier to find

Part 5 — First Launch on Windows

  1. Open VMware Workstation Pro from the desktop shortcut or Start Menu.
  2. Accept the End User License Agreement if prompted.
  3. If asked for a license key, choose the free-use option instead of entering a key.
  4. Confirm that the main VMware Workstation Pro window opens.

Part 6 — Verify VMware Network Adapters

  1. Open VMware Workstation Pro.
  2. Go to:
Edit > Virtual Network Editor
  1. Look for:
VMnet1 Host-only
  1. Confirm that VMnet1 exists.
  2. Do not use Bridged networking for class security labs unless specifically instructed.
Recommended Windows lab network: VMnet1 Host-only

3. Installing VMware Fusion Pro on macOS

Product to Install

MacBook students should install:

VMware Fusion Pro

Important Mac Note: Intel vs. Apple Silicon

Mac Type Processor Type VM Requirement
Intel MacBook Intel x86_64 Use x86_64 virtual machine images
Apple Silicon MacBook M1, M2, M3, M4, or later Use ARM64 virtual machine images
Architecture warning: Apple Silicon Macs cannot run traditional x86_64 VM images in VMware Fusion. Intel Macs cannot run ARM64 VM images. Students must download the correct Kali and Ubuntu images for their Mac type.

Part 1 — Check the Mac Type

  1. Click the Apple menu.
  2. Select About This Mac.
  3. Look for one of the following:
What You See Meaning VM Image Type
Processor: Intel Intel Mac x86_64
Chip: Apple M1/M2/M3/M4 Apple Silicon Mac ARM64

Part 2 — Check macOS System Requirements

Before installation, confirm the following:

  • The Mac is running a supported version of macOS.
  • The student has administrator access.
  • The Mac has enough memory and disk space for two virtual machines.
Resource Recommended Minimum Workable
RAM 16 GB or more 8 GB
Free disk space 80–100 GB 50 GB
Permissions Administrator account Administrator password available

Part 3 — Download VMware Fusion Pro

  1. Open a web browser.
  2. Go to the Broadcom Support Portal.
  3. Sign in with a Broadcom account. Create a free account if needed.
  4. Search for:
VMware Fusion Pro
  1. Select the latest available version of VMware Fusion Pro.
  2. Accept any required terms and conditions.
  3. Download the .dmg installer file.

Part 4 — Install VMware Fusion Pro

  1. Open Finder.
  2. Go to the Downloads folder.
  3. Double-click the downloaded VMware Fusion .dmg file.
  4. When the installer window opens, either:
    • Double-click the VMware Fusion icon, or
    • Drag the VMware Fusion icon into the Applications folder.

Part 5 — Approve macOS Security Prompts

macOS may show a message similar to:

VMware Fusion is an application downloaded from the Internet.
Are you sure you want to open it?

Click:

Open

Enter the Mac administrator password if prompted.

Part 6 — Complete VMware Fusion Setup

  1. Open VMware Fusion from the Applications folder.
  2. Accept the license agreement.
  3. If prompted for a license key, choose the free-use option rather than entering a license key.
  4. Allow VMware Fusion to initialize.
  5. Approve required macOS permissions if prompted.

Possible macOS permission prompts include:

  • System Extension
  • Network access
  • Removable volume access
  • Accessibility
  • Full Disk Access

Part 7 — Confirm VMware Fusion Is Installed

  1. Open Finder.
  2. Go to Applications.
  3. Confirm that VMware Fusion appears.
  4. Launch VMware Fusion.
  5. Confirm that the main VMware Fusion window opens.

Part 8 — Recommended VMware Fusion Network Setting

For the Network Security labs, configure each VM to use:

Private to my Mac

Do not use the following setting unless the instructor specifically authorizes it:

Bridged Networking
Recommended macOS lab network: Private to my Mac

4. Post-Installation Setup for Both Windows and macOS

Step 1 — Create a Folder for Virtual Machines

Recommended folder structure:

Windows

C:\VMs\Network-Security\

macOS

Documents/VMs/Network-Security/

Step 2 — Download the Correct Virtual Machine Images

Students will usually need:

Kali Linux
Ubuntu Desktop or Ubuntu Server
Host Computer Recommended VM Architecture
Windows with Intel/AMD CPU x86_64
Intel Mac x86_64
Apple Silicon Mac ARM64

Step 3 — Verify VMware Opens Correctly

  1. Launch VMware Workstation Pro or VMware Fusion Pro.
  2. Confirm that the application opens without errors.
  3. Confirm that you can create or import a virtual machine.

Step 4 — Prepare for Lab 1.1

Before starting the first virtual lab, students should have:

  • VMware installed.
  • Kali Linux image downloaded.
  • Ubuntu image downloaded.
  • At least 80–100 GB of free storage available.
  • The correct VM architecture for their computer.

5. Network Settings for the Labs

Host OS VMware Product Recommended Lab Network Use Case
Windows VMware Workstation Pro VMnet1 Host-only Isolated Kali–Ubuntu lab communication
macOS Intel VMware Fusion Pro Private to my Mac Isolated Kali–Ubuntu lab communication
macOS Apple Silicon VMware Fusion Pro Private to my Mac Isolated Kali–Ubuntu lab communication

Do Not Use Bridged Networking for These Labs

Warning: Bridged networking can place the virtual machine on the same network as the host computer. For cybersecurity labs, this can expose the VM to non-lab systems and may create safety or policy issues.

Temporary Internet Access

Some labs may require installing packages inside Kali or Ubuntu. When Internet access is needed temporarily, use NAT/shared networking, install the package, and then return to the private lab network.

Host Platform Temporary Internet Setting Return to Lab Setting
Windows VMware Workstation Pro NAT VMnet1 Host-only
macOS VMware Fusion Pro Share with my Mac Private to my Mac

7. Troubleshooting

Problem Likely Cause Suggested Fix
VMware installer will not run on Windows Installer was not run as administrator Right-click the installer and choose Run as administrator.
VMware asks for a license key Free-use option was not selected Choose the free/personal-use option instead of entering a key.
Download is blocked or unavailable Broadcom account or profile issue Complete the required account profile, sign-in, and download agreement steps.
VMware Fusion will not open on macOS macOS security or permission prompt Open VMware Fusion from Applications and approve required permissions.
Apple Silicon Mac cannot run downloaded VM Wrong VM architecture Use ARM64 Kali and Ubuntu images.
Intel Mac cannot run ARM image Wrong VM architecture Use x86_64 Kali and Ubuntu images.
VMs cannot communicate Wrong network setting Use VMnet1 Host-only on Windows or Private to my Mac on macOS.
VM has no Internet Host-only/private network is isolated Temporarily switch to NAT or Share with my Mac, then switch back.
VM is slow Not enough RAM or CPU resources Close other apps and reduce VM resource usage.
Port scanning labs show unexpected systems Wrong network mode, possibly bridged networking Stop the lab and confirm the VM is not bridged.
