1. Lab Purpose

The purpose of this lab is to teach students how input validation reduces web application risk. Students will create a simple local web application with weak input handling and improved input validation. They will compare how the application responds to valid and invalid input, then document validation rules, evidence, and recommendations.

Main goal: Students should understand that input validation is a server-side security control that checks whether user input is expected, reasonable, and safe before the application processes it.

2. Learning Objectives

  1. Define input validation and explain why it is important for web application security.
  2. Distinguish client-side validation from server-side validation.
  3. Build and run a local Flask input-validation training application.
  4. Identify weak input handling in a simple web form.
  5. Implement server-side validation rules for name, email, age, quantity, and comments.
  6. Use allowlists, type checks, length limits, and range checks.
  7. Test the application with valid, invalid, boundary, and special-character inputs.
  8. Review application logs for rejected and accepted inputs.
  9. Create a validation test matrix and remediation plan.
  10. Explain OS-specific differences for Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu.

3. Lab Scenario

A small organization is developing an internal request form. The first version accepts nearly anything users submit. Your task is to test the weak version, improve the validation logic, and show how server-side validation changes the application's behavior. You must keep all testing local and avoid real sensitive data.

7. Key Concepts

Input Validation

Input validation checks user-supplied data before the application processes it. Good validation asks whether the data is expected, properly formatted, within an allowed range, and appropriate for the business function.

Client-Side vs. Server-Side Validation

Client-side validation improves user experience but can be bypassed. Server-side validation is required because the server must not trust input simply because it came from a browser form.

Allowlist Validation

Allowlist validation defines what is acceptable and rejects everything else. This is usually stronger than trying to block every possible bad pattern.

Boundary Testing

Boundary testing checks values near the edge of allowed limits, such as minimum age, maximum quantity, or maximum comment length.

Validation ControlPurposeLab Example
Required field checkReject missing required valuesName and email cannot be blank
Type checkConfirm expected data typeAge and quantity must be numbers
Range checkLimit numeric valuesAge 13–120; quantity 1–20
Length limitPrevent unreasonable input sizeComment limit of 200 characters

Part 5 — Create the Local Input Validation App

On Ubuntu-Target, create the application file:

cat > ~/Lab5-2-InputValidation/app/app.py <<'EOF'
from flask import Flask, request, jsonify, make_response
import html
import logging
import re

app = Flask(__name__)
logging.basicConfig(
    filename="input_validation_lab.log",
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)

DEPARTMENTS = {"IT", "HR", "Finance", "Operations"}

def page(title, body):
    return make_response(
        "<!DOCTYPE html><html><head><title>" + html.escape(title) +
        "</title></head><body><h1>" + html.escape(title) + "</h1>" +
        body + "<hr><p><a href='/'>Home</a></p></body></html>"
    )

def validate_submission(data):
    errors = []

    name = str(data.get("name", "")).strip()
    email = str(data.get("email", "")).strip()
    department = str(data.get("department", "")).strip()
    age_raw = str(data.get("age", "")).strip()
    quantity_raw = str(data.get("quantity", "")).strip()
    comment = str(data.get("comment", "")).strip()

    if not name:
        errors.append("Name is required.")
    elif not re.fullmatch(r"[A-Za-z][A-Za-z .'-]{1,49}", name):
        errors.append("Name must be 2-50 characters and use only letters, spaces, periods, hyphens, or apostrophes.")

    if not email:
        errors.append("Email is required.")
    elif not re.fullmatch(r"[^@\s]+@[^@\s]+\.[^@\s]+", email):
        errors.append("Email format is invalid.")

    if department not in DEPARTMENTS:
        errors.append("Department must be one of: IT, HR, Finance, Operations.")

    if not age_raw.isdigit():
        errors.append("Age must be a whole number.")
    else:
        age = int(age_raw)
        if age < 13 or age > 120:
            errors.append("Age must be between 13 and 120.")

    if not quantity_raw.isdigit():
        errors.append("Quantity must be a whole number.")
    else:
        quantity = int(quantity_raw)
        if quantity < 1 or quantity > 20:
            errors.append("Quantity must be between 1 and 20.")

    if len(comment) > 200:
        errors.append("Comment must be 200 characters or fewer.")

    return errors

@app.route("/")
def home():
    body = """
    <ul>
      <li><a href="/weak-form">Weak form</a></li>
      <li><a href="/validated-form">Validated form</a></li>
      <li><a href="/logs-summary">Logs summary</a></li>
    </ul>
    <p>This app is for local defensive input-validation training only.</p>
    """
    return page("Lab 5.2 Web Application Input Validation", body)

@app.route("/weak-form", methods=["GET", "POST"])
def weak_form():
    if request.method == "POST":
        data = request.form.to_dict()
        logging.info("WEAK_ACCEPTED %r", data)
        body = "<h2>Weak Form Accepted Input Without Validation</h2><pre>" + html.escape(str(data)) + "</pre>"
        return page("Weak Form Result", body)
    return page("Weak Form", form_html("/weak-form"))

@app.route("/validated-form", methods=["GET", "POST"])
def validated_form():
    if request.method == "POST":
        data = request.form.to_dict()
        errors = validate_submission(data)
        if errors:
            logging.warning("VALIDATED_REJECTED data=%r errors=%r", data, errors)
            body = "<h2>Validated Form Rejected Input</h2><ul>" + "".join(["<li>" + html.escape(e) + "</li>" for e in errors]) + "</ul>"
            return page("Validated Form Rejected", body), 400
        logging.info("VALIDATED_ACCEPTED %r", data)
        body = "<h2>Validated Form Accepted Input</h2><pre>" + html.escape(str(data)) + "</pre>"
        return page("Validated Form Accepted", body)
    return page("Validated Form", form_html("/validated-form"))

@app.route("/api/weak-submit", methods=["POST"])
def api_weak_submit():
    data = request.get_json(silent=True) or {}
    logging.info("API_WEAK_ACCEPTED %r", data)
    return jsonify({"status": "accepted_without_validation", "received": data})

@app.route("/api/validated-submit", methods=["POST"])
def api_validated_submit():
    data = request.get_json(silent=True) or {}
    errors = validate_submission(data)
    if errors:
        logging.warning("API_VALIDATED_REJECTED data=%r errors=%r", data, errors)
        return jsonify({"status": "rejected", "errors": errors}), 400
    logging.info("API_VALIDATED_ACCEPTED %r", data)
    return jsonify({"status": "accepted", "received": data})

@app.route("/logs-summary")
def logs_summary():
    try:
        with open("input_validation_lab.log", "r") as f:
            lines = f.readlines()[-30:]
    except FileNotFoundError:
        lines = ["No log entries yet."]
    return page("Logs Summary", "<pre>" + html.escape("".join(lines)) + "</pre>")

def form_html(action):
    return f"""
    <form method='post' action='{action}'>
      <p>Name: <input name='name'></p>
      <p>Email: <input name='email'></p>
      <p>Department: <input name='department' placeholder='IT, HR, Finance, Operations'></p>
      <p>Age: <input name='age'></p>
      <p>Quantity: <input name='quantity'></p>
      <p>Comment: <input name='comment' size='60'></p>
      <p><button type='submit'>Submit</button></p>
    </form>
    """

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5002)
EOF

15. Troubleshooting Guide

ProblemLikely CauseFix
Kali cannot reach the appWrong IP, Flask not running, or firewall blocking port 5002Check ip -brief addr, ss -tulpen, and UFW status.
python3 -m venv failspython3-venv package missingAsk instructor, then install python3-venv if allowed.
Validated form accepts bad inputValidation rule missing or test value was not actually invalidReview validate_submission() and test values.
Validated form rejects valid inputRule is too strictAdjust the validation rule and explain the decision.
JSON endpoint returns unsupported media typeMissing Content-Type: application/jsonAdd the header to the curl command.
Log file is emptyNo requests made yet or wrong directorySend requests, then check ~/Lab5-2-InputValidation/app/input_validation_lab.log.
scp failsWrong username, IP, path, or SSH issueVerify SSH access and exact evidence folder path.
