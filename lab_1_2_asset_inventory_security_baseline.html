1. Lab Purpose

The purpose of this lab is to build a repeatable security baseline for the virtual lab environment created in Lab 1.1. Students will collect system, network, account, service, package, firewall, and update information from both Kali-Lab and Ubuntu-Target.

Main goal: Students should understand that an asset inventory and baseline help defenders know what systems exist, what is running, what is exposed, and what changes over time.

2. Learning Objectives

  1. Explain the purpose of asset inventory in network security.
  2. Collect hostnames, IP addresses, operating system versions, and architecture information.
  3. Document local user accounts and groups on Ubuntu and Kali.
  4. Identify listening services and open local ports.
  5. Record firewall status and basic firewall posture.
  6. Collect installed package and update-status information.
  7. Create a repeatable baseline folder structure for evidence.
  8. Compare baseline findings between Kali-Lab and Ubuntu-Target.
  9. Prepare a concise baseline report for future comparison.
  10. Explain Windows, Intel Mac, Apple Silicon Mac, Kali, and Ubuntu differences.

3. Lab Scenario

You are a new security analyst assigned to document the initial state of a small virtual lab network. Before future scanning, firewall, VPN, cryptography, or monitoring labs are performed, you must create a baseline inventory of each system. This baseline will help you identify later changes, misconfigurations, and unexpected services.

6. Safety Rules

  1. Inventory only your assigned Kali-Lab and Ubuntu-Target VMs.
  2. Do not scan or inventory public systems, home routers, school networks, or classmates’ machines.
  3. Do not submit passwords, private keys, or sensitive personal information.
  4. Do not change firewall, service, or user settings unless the lab explicitly instructs you to do so.
  5. Use host-only/private networking unless the instructor authorizes temporary Internet access.
  6. When collecting user information, document account names only; do not collect password hashes.
Important: This lab is for inventory and documentation only. It is not a vulnerability exploitation lab.

7. Key Concepts

Asset Inventory

An asset inventory is a structured list of systems, services, software, users, and configuration details. Security teams use inventories to understand what must be protected.

Security Baseline

A security baseline records the known-good or starting condition of a system. Later changes can be compared against the baseline to detect drift, misconfiguration, or unexpected exposure.

11. OS-Specific Differences

Kali-Lab

Kali is used as the security workstation. It stores the final report, runs selected Nmap checks, and collects Ubuntu evidence.

ip -brief addr
hostnamectl
dpkg -l | grep -E 'nmap|wireshark|tcpdump|curl|openssh'
nmap -sV -p 22,80,443,8080 UBUNTU-IP-ADDRESS
scp -r user@UBUNTU-IP:/path/to/evidence .

Ubuntu is the target/server VM. Most baseline evidence is collected locally from Ubuntu.

15. Troubleshooting Guide

ProblemLikely CauseFix
Kali cannot ping UbuntuWrong IP or VMs are on different networksVerify ip -brief addr and host-only/private network settings.
scp failsWrong username, IP address, path, or SSH issueTest SSH login first and verify the evidence folder path.
ufw command not foundUFW is not installedDocument that UFW was unavailable, or install it only if instructor permits.
resolvectl does not workDifferent Linux configurationUse cat /etc/resolv.conf instead.
apt update failsLab network is isolatedDocument that the update check was not performed because isolation was required.
Nmap shows fewer ports than ssService may be bound to localhost or filtered by firewallCompare local listener address and firewall state.
Too much package outputdpkg -l is longSubmit the saved file and summarize only key packages in the report.
